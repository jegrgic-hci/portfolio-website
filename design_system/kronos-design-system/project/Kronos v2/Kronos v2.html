Protocol_V5.0 · Build 5.0.0 · 17 May 2026
KRONOS_LAB v2 proposal

Same lab/protocol DNA — mono UI chrome, hard 1px edges, teal accent. The fixes target three real problems: greys that failed WCAG, type that ran too small and over-tracked, and a component set that stopped at the portfolio. v2 keeps every component editable as the same atoms; only the tokens, sizes, and accessibility wiring change.

The biggest single issue in v1. Three foreground greys were calibrated by eye, not by ratio. --k-fg-3 (used for captions, helper text, sidebar muted items) failed AA outright. v2 darkens everything and adds an explicit fourth step for hint-only text.

03

Typography

Two changes: Anton replaces Impact (Impact ships inconsistently on Linux and renders differently on every OS), and all-caps mono is restricted to short labels — buttons, tags, nav items, version stamps. Headings, table cells, helper text and form labels switch to sentence-case Plex Sans, which doubles scanning speed without losing the lab/protocol voice.

  • Mono survives where it pays. Buttons, tags, tabs, sidebar nav, version stamps stay uppercase Plex Mono.
  • Sentence case where it costs. Headings, labels, helper text, table cells switch to Plex Sans — these are the strings users read, not the chrome they tap.

Responsive reflow

v1 was a single-breakpoint design — 280px fixed sidebar, 900px content max, 96px padding. v2 keeps the layout vocabulary but defines three states: desktop ≥ 1024px (sidebar open), tablet 640–1024px (sidebar collapses to icon rail), mobile < 640px (drawer + bottom-nav fallback). Below is the desktop frame; resize the browser to see it react.

  • Sidebar collapses to 56px icon rail at the tablet breakpoint; full text returns at desktop.
  • Drawer pattern at mobile — full-screen overlay, focus trap, ESC to close.
  • Fluid padding. Page padding scales from 96px → 24px via clamp().
  • Project grid goes 3-col → 2-col → 1-col at the same breakpoints.

What comes next

  • Migrate existing apps by swapping colors_and_type.css for tokens-v2.css. The token names are --k2-* so v1 and v2 can coexist while you migrate component by component.
  • Decide whether you want a dark mode pass next; v2 has the contrast headroom for it but the inverse tokens aren't fully fleshed out yet.
  • Add a data-table primitive — the next obvious gap after dropdown / modal / banner. Worth a small follow-up.
  • Document a real product example (form + table + side panel) so the new components are seen in flow, not just as isolated specimens.
