1-Click opt-in flow
From SMA overview screen, seller clicks to add discount (preset)
After adding discount, confirmation noted via text change and snackbar at bottom of scren
Overall research question for this flow
What are some questions specific to this flow that will impact the design and launch of this feature?
First screen
Where would you go to enable this feature? Is it clear what this feature does? Based on what you read, how likely would you be to enable? Follow up why/why not Do you understand where this discount is coming from? Are there any concerns about enabling this feature? What would you do if you wanted to change the discount amount? What would you expect to happen if you 'turned on" this feature here?
Second Screen (hypothetically turned on)
Does this align with your expectations? why/why not? If you were to undo or make a change, what would you do?
Follow up questions
Based on this flow, how easy would you say this was? How satisfied this feature meets your needs? If this did not exist on this screen, where would you expect to find this feature within SMA? If this feature was to go live, would you want to receive an email or notified of this? why/why not?
